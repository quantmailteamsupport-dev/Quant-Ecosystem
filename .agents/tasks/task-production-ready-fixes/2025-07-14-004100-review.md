## Auth hook reply guard (v1 Issue #1 â€” resolved)

The `onRequest` hook in `packages/server-core/src/app.ts` now reads:

```typescript
await fastify.requireAuth()(request, reply);
if (reply.sent) return;
```

The test suite was updated to assert that unknown routes return 401 without a token and 404 with a valid token, confirming the global enforcement works end-to-end.

## Public-path prefix matching

The auth bypass list checks `path === p || path.startsWith(p + '/')`. This means `/healthz/foo` or `/metrics/admin` would skip auth. No routes are currently registered under those prefixes, so this is not exploitable today. However, if a monitoring dashboard or Prometheus scrape endpoint is later mounted at `/metrics/targets`, it would inherit the auth bypass. Switching to exact-match (`publicPaths.includes(path)`) would be tighter with no downside since health and metrics endpoints don't use sub-paths.
