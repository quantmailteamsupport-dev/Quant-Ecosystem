# Production-readiness fixes across Quant Ecosystem

A broad stabilization pass that fixes runtime crashes, port mismatches, build errors, and introduces global auth enforcement. The highest-impact change is the Prisma plugin (`packages/server-core/src/plugins/prisma.ts`) that decorates every Fastify instance with a `PrismaClient`, resolving the crash when any route called `fastify.prisma`. A global `onRequest` hook now enforces JWT auth on all routes except health/metrics paths. Frontend proxies in QuantAI and QuantChat had hardcoded ports pointing to the wrong backends; these are corrected. The `@quant/ai` package is updated for Vercel AI SDK v5 type changes. Conflicting Next.js pages/ files are removed from apps that use app/ routing.

**Watch for:** The global auth hook calls `requireAuth()` but does not send a reply when token validation fails on the `onRequest` path â€” it relies on `requireAuth()` internally calling `reply.send()`, which halts the Fastify lifecycle via an implicit early return. If that behavior changes in a Fastify upgrade, unauthenticated requests could fall through to route handlers (confirmed). The `as any` casts on the AI SDK model instances suppress type safety and will silently break if the model interface changes (confirmed). The `@quant/database` tsconfig path mappings now point to `dist/` which means a build step must run before typechecking the consuming apps â€” CI pipelines need to enforce this order (confirmed).

## High-level view

The Prisma plugin is thin glue: it re-exports the singleton `prisma` instance from `@quant/database` onto the Fastify instance and disconnects on shutdown. It fixes the immediate crash but doesn't add connection health monitoring, which means the health endpoint won't detect a dead database connection.

Global auth enforcement is a single `onRequest` hook that gates every non-health route through `requireAuth()`. The design delegates reply construction to the auth plugin's closure rather than the hook itself, making correctness dependent on the auth plugin's internal response behavior. The prefix matching on public paths (`path.startsWith(p + '/')`) is more permissive than necessary â€” `/healthz/anything` would bypass auth.

Port corrections are straightforward constant changes: QuantAI proxy routes move from 3002â†’3020, QuantChat API proxies from 3108â†’3002, and WebSocket URLs from 3001â†’3002. The QuantChat `next.config.js` gains a webpack override to externalize Node.js-only packages from the client bundle, paired with a new `@quant/realtime/client` entry point that only exports browser-safe symbols.

The AI SDK changes rename `maxTokens` â†’ `maxOutputTokens` and adapt usage-tracking to the v5 response shape where `promptTokens`/`completionTokens` became `inputTokens`/`outputTokens`. Both `generateText` and `streamText` calls cast the model argument to `any` to suppress provider type mismatches.

Conflicting pages/ directory files are deleted wholesale. The `quantads` campaign detail page body is replaced with `return null`, leaving a routable page that renders blank rather than being deleted or redirected.

## Prisma plugin and database lifecycle

The plugin at `packages/server-core/src/plugins/prisma.ts` imports the singleton `prisma` from `@quant/database` and decorates it onto the Fastify instance. The `onClose` hook disconnects the client. However, the health plugin's `/readyz` route only checks Redis connectivity â€” there's no database liveness probe. In production, Prisma's connection pool could exhaust or the database could become unreachable without the readiness endpoint reflecting that state.

The `@quant/database` package exposes `main: "dist/index.js"` and `types: "dist/index.d.ts"`, which is why the tsconfig path mappings now point to `dist/`. This means the consuming app's typecheck fails if the database package hasn't been compiled. The previous mapping (`src/index.ts`) worked for development iteration without a build step; the new mapping trades that convenience for correctness (the generated Prisma types only exist in `dist/` after `prisma generate && tsc`).

## Global auth enforcement design

```
onRequest hook
  â””â”€ path matches public list? â†’ skip
  â””â”€ else â†’ requireAuth()(request, reply)
```

The `requireAuth()` factory returns a preHandler-shaped function that calls `reply.status(401).send(...)` when the token is missing or invalid. But the hook is an `onRequest` hook, not a preHandler â€” the lifecycle semantics differ. In `onRequest`, returning from an async function after calling `reply.send()` does halt processing in Fastify v5, but this is an implicit contract. A defensive pattern would be:

```typescript
await fastify.requireAuth()(request, reply);
if (reply.sent) return;
```

Without that guard, a future change to the auth plugin that throws instead of sending a reply would let requests pass through unauthenticated.

The prefix matching (`path.startsWith(p + '/')`) on public paths means `/healthz/anything` or `/metrics/secret-data` would bypass auth. Exact-match only would be tighter.

## AI SDK v5 type adaptation

The engine now uses fallback chains to handle both old and new usage field names:

```typescript
(response.usage as any)?.promptTokens ?? response.usage?.inputTokens ?? fallback;
```

This is useful during a migration window but leaves dead code paths once the SDK version is pinned. The `as any` casts on the model parameter suppress the type mismatch between the provider's return type and what `generateText`/`streamText` expect â€” a known Vercel AI SDK ergonomic issue where provider types don't unify cleanly. The casts should be revisited when the SDK stabilizes its type exports.

## Deleted pages and routing cleanup

QuantAds deletes `analytics.tsx`, `audiences.tsx`, `campaigns.tsx`, and `index.tsx` from `src/pages/`, and QuantChat deletes `camera.tsx`, `map.tsx`, and `stories.tsx`. The campaign detail page (`pages/campaign/[id].tsx`) isn't deleted but its body is replaced with `return null` â€” this leaves a routable page that renders empty. It would be cleaner to delete it entirely or redirect to the app/ equivalent.
