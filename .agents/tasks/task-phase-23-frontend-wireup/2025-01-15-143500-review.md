# React Query wire-up across 7 frontend apps (v3)

All seven Quant Ecosystem frontend apps now fetch data through typed React Query hooks instead of hardcoded mock generators. The four issues from the v2 review are resolved: `useFeed` passes `offset` to paginate correctly, `fetchNextPage` is wrapped in a `useEffect`, `useLive` exposes fetched stream data through `viewers`/`topGifters`, and the `as any` cast in `useReels` is gone. Zero `mock`/`FIXME(phase-23)` references remain in `.tsx` files.

Watch for:

- **`useLive` topGifters derives "diamonds" from viewer counts** (confirmed) — a semantic mismatch that renders misleading leaderboard data, but functionally harmless until a real gifting API exists.
- **Live page ignores fetched stream list** (confirmed) — `live.tsx` doesn't destructure `streams` from the hook, unconditionally rendering an empty state even when stream data is available.

**Verdict**: APPROVED

## High-level view

The v2 fix commit addresses all four flagged issues. `apiClient.getForYouFeed` accepts an `offset` parameter, and the auto-prefetch logic lives in a `useEffect` with correct dependencies rather than executing during render. `useLive` now derives `viewers` and `topGifters` from the fetched stream list.

Two minor UI-layer gaps remain. The live page doesn't consume the `streams` array the hook now exposes, and the "top gifters" leaderboard repurposes viewer counts as diamond amounts. Neither blocks shipping; both are one-line follow-ups.

The hook pattern across all 7 apps is uniform: `useQuery`/`useInfiniteQuery` for reads, `useMutation` with `invalidateQueries` for writes, consistent error extraction from the `ApiResponse<T>` envelope, and `queryKey` arrays that scope cache correctly. The `QueryClient` configuration (30s stale, 3 retries, no refetch-on-focus) is identical everywhere.

## useLive data exposure gap on the page layer

The hook now returns `streams: LiveEvent[]` alongside `viewers` and `topGifters` derived from that array. The page at `live.tsx` destructures:

```ts
const {
  stream,
  chat,
  topGifters,
  isStreaming,
  startStream,
  endStream,
  updateSettings,
  sendChat,
  settings,
} = useLive('current-user');
```

`streams` is not destructured. The "Live Now" section unconditionally shows `<EmptyState title="No live streams" .../>` regardless of what the API returns. This is a one-line destructuring fix followed by a conditional render.

The `topGifters` field uses `viewerCount` as the `diamonds` value, which means the gifter leaderboard during a live stream displays viewer counts (e.g., "3,420") under a diamond icon. Acceptable as a placeholder until a dedicated gifting endpoint ships.

## Redundant useEffect in challenges page

`challenges.tsx` calls `loadChallenges()` in a mount-only `useEffect`, but React Query already fetches on mount by default. The `loadChallenges` function just calls `challengesQuery.refetch()`, resulting in a redundant request on mount. Not a bug — React Query's deduplication window prevents duplicate network calls — but the `useEffect` can be removed entirely.
