## CommandMenu handleKeyDown closure concern

The `handleKeyDown` callback includes `filteredCommands` in its dependency array:

```ts
const handleKeyDown = useCallback(
  (e: React.KeyboardEvent) => {
    if (e.key === 'ArrowDown') {
      setActiveIndex((i) => Math.min(i + 1, filteredCommands.length - 1));
    }
    // ...
  },
  [filteredCommands, activeIndex, onClose],
);
```

The edge case: a user holds ArrowDown while typing, shrinking the list. The `onChange` handler resets `setActiveIndex(0)` on every keystroke, but if the ArrowDown event fires in the same tick before React re-renders, `filteredCommands.length` in the closure is stale (larger than the new list). The `Math.min` clamp could set `activeIndex` to an out-of-bounds value for one frame, producing a momentary highlight on a non-existent item. Corrected on next render when the dep array triggers a new memoization.
