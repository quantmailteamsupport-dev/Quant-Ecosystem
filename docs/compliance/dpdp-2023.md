# Digital Personal Data Protection Act 2023 (India)

## Overview

The DPDP Act 2023 governs the processing of digital personal data in India. It applies to the Quant platform when processing data of Indian residents (Data Principals).

## Key Definitions

- **Data Principal**: Individual whose personal data is processed
- **Data Fiduciary**: Entity determining purpose and means of processing (Quant)
- **Data Processor**: Entity processing data on behalf of the fiduciary
- **Consent Manager**: Registered entity managing consent on behalf of principals

## Data Principal Rights

### Right to Access Information (Section 11)

- Summary of personal data being processed
- Processing activities performed
- Identity of other data fiduciaries with shared data
- **Implementation**: Privacy dashboard with data summary view

### Right to Correction and Erasure (Section 12)

- Correction of inaccurate or misleading data
- Completion of incomplete data
- Erasure of data no longer necessary for stated purpose
- **Implementation**: Self-service correction tools, deletion API

### Right to Grievance Redressal (Section 13)

- Grievance mechanism with designated officer
- Response within prescribed timeframe
- Escalation path to Data Protection Board
- **Implementation**: In-app grievance form, ticketing system

### Right to Nominate (Section 14)

- Nominate representative to exercise rights in case of death/incapacity
- **Implementation**: Nomination registration in account settings

## Data Fiduciary Obligations

### Notice and Consent (Section 5-6)

- Clear, specific notice before collection
- Consent freely given, specific, informed, unambiguous
- Purpose limitation enforced
- **Implementation**: Consent collection UI with granular purpose selection

### Purpose Limitation (Section 4)

- Data processed only for consented purpose
- New purposes require fresh consent
- **Implementation**: Purpose tracking in data processing pipeline

### Data Accuracy (Section 8(3))

- Ensure completeness, correctness, consistency
- Reasonable effort to maintain accuracy
- **Implementation**: Validation rules, user verification flows

### Data Security (Section 8(4))

- Reasonable security safeguards
- Protect against breach
- **Implementation**: Encryption, access controls, monitoring

### Data Retention (Section 8(7))

- Retain only as long as necessary for purpose
- Erase upon purpose fulfillment or consent withdrawal
- **Implementation**: Automated retention enforcement

### Breach Notification (Section 8(6))

- Notify Data Protection Board and affected principals
- Notification "without unreasonable delay"
- **Implementation**: Automated breach detection and notification workflow

## Significant Data Fiduciary (Section 10)

Additional obligations if designated as Significant Data Fiduciary:

- Appoint Data Protection Officer (India-based)
- Appoint independent Data Auditor
- Conduct periodic Data Protection Impact Assessment
- **Status**: Assessment pending based on volume thresholds

## Cross-Border Transfer (Section 16)

- Transfer permitted except to notified restricted countries
- Government may restrict transfer to specific countries
- **Implementation**: Data residency controls, transfer logging

## Children's Data (Section 9)

- Verifiable consent from parent/guardian for data of persons under 18
- No behavioral monitoring or targeted advertising directed at children
- **Implementation**: Age gate, parental consent flow

## Consent Manager Integration

- Registered Consent Managers can manage consent on behalf of principals
- Interoperability with Consent Manager APIs
- **Implementation**: Consent Manager adapter interface

## Penalties

- Up to INR 250 crore per instance of violation
- Penalties determined by Data Protection Board
- Compliance program designed to demonstrate good faith
